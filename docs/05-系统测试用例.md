# 第五章 系统测试

> 本章给出针对 AI 智能题库系统的完整系统测试用例，覆盖 **功能测试、安全测试、兼容性测试、性能测试、易用性测试** 五大维度。所有用例均基于系统已实现的 22 个 REST 接口 + 10 张数据库表 + 9 个前端页面设计，可在本地环境（`start.ps1` 启动后）直接执行。

---

## 5.1 测试概述

### 5.1.1 测试目标

| 维度 | 目标 | 关键指标 |
|---|---|---|
| **功能** | 业务流程完整、数据一致、状态流转正确 | 用例通过率 ≥ 95% |
| **安全** | 未授权访问拦截、SQL 注入/XSS 防御、密码非明文 | 攻击向量全部失败 |
| **兼容** | 多浏览器、多分辨率、前后端协同正常 | 主流浏览器全部通过 |
| **性能** | 接口响应、并发支撑、大数据量不卡顿 | P95 < 500ms、并发 50 不掉链 |
| **易用** | UI 友好、错误提示清晰、流程直观 | 关键路径 5 分钟内可上手 |

### 5.2.1 认证模块（3 个接口）

#### TC-F-001 用户注册

| 项 | 内容 |
|---|---|
| **用例编号** | TC-F-001 |
| **模块** | 认证 |
| **接口** | `POST /api/auth/register` |
| **优先级** | P0 |
| **前置条件** | 数据库 `users` 表已建，密码 BCrypt 加密 |
| **测试步骤** | 1. POST `/api/auth/register`，Body：`{"username":"test001","password":"pwd123","role":"STUDENT","realName":"测试用户","email":"test001@xx.com"}` <br> 2. 观察返回结构 <br> 3. 用相同账号再次注册 |
